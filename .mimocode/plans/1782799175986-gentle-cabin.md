## Phase 2: Dashboard Layout Responsif

### 2.1 Sidebar Responsif
**File:** `src/app/(dashboard)/layout.tsx`
- **Mobile (< lg):** Hamburger menu → Sheet sidebar (sudah ada, perlu perbaikan)
- **Tablet (lg-xl):** Sidebar collapsible (icon saja, expand on hover)
- **Desktop (> xl):** Sidebar full width tetap
- Fix: Sheet sidebar mobile harus full-height, ada overlay yang benar, animasi smooth

### 2.2 Header Bar Mobile
- Tambahkan header bar di mobile: logo + hamburger + user avatar
- Di desktop: header minimal (hanya breadcrumb atau judul halaman)

### 2.3 Main Content Padding
- Mobile: padding lebih kecil (`p-3` atau `p-4`)
- Desktop: padding standar (`p-6`)
- Gunakan responsive classes: `p-3 sm:p-4 lg:p-6`

## Phase 3: Tabel Responsif

### 3.1 Strategy: Horizontal Scroll + Card View Toggle
**Heuristik:** Flexibility & Efficiency (#7), Aesthetic (#8)
**File:** Semua file tabel

**Mobile (< sm):**
- Tabel dikonversi ke **Card View** — setiap baris jadi card terpisah
- Setiap field jadi label-value pair dalam card
- Tombol aksi di bawah card

**Tablet/Desktop (≥ sm):**
- Tabel horizontal scroll (sudah ada dari shadcn `overflow-x-auto`)
- Kolom kunci selalu terlihat: No, Nama/Info Utama, Aksi
- Kolom opsional: scrollable

### 3.2 Implementasi per Halaman
| Halaman | Kolom Utama (Mobile) | Kolom Sembunyi (Mobile) |
|---|---|---|
| Barang | Kode, Nama, Stok, Status | Kategori, Harga, Aksi Detail |
| Kategori | Nama, Aksi | - |
| Supplier | Nama, Telepon, Aksi | Email, Alamat |
| Barang Masuk | No Transaksi, Barang, Jumlah | Tanggal, Supplier |
| Barang Keluar | No Transaksi, Barang, Jumlah | Tanggal, Tujuan |
| Users | Nama, Email, Status | Role, Aksi |
| Laporan | Sesuai tab aktif | - |

## Phase 4: Form Responsif

### 4.1 Form Grid Layout
**Heuristik:** Consistency (#4), Aesthetic (#8)
**File:** Semua form dialog

**Mobile (< sm):**
- Semua field full-width (stack vertical)
- Dialog full-width (max-w-[calc(100%-1rem)])
- Tombol submit full-width

**Desktop (≥ sm):**
- Grid 2 kolom untuk field pendek (kode, nama, satuan, harga)
- Grid 1 kolom untuk field panjang (alamat, tujuan)
- Dialog max-w-md atau max-w-lg

### 4.2 Touch-Friendly
- Minimum touch target 44x44px untuk button di mobile
- Input field height minimal 44px di mobile (`h-11 sm:h-9`)
- Select dropdown harus mudah di-tap

## Phase 5: Heuristik Spesifik

### 5.1 Error Prevention — Konfirmasi Transaksi
**File:** `barang-keluar/page.tsx`, `barang-masuk/page.tsx`
- Tambahkan summary dialog sebelum submit:
  ```
  Barang: [Nama] (Stok: [X])
  Jumlah: [Y]
  Sisa Stok: [X-Y]
  ```
- Tombol "Konfirmasi" dan "Batal"

### 5.2 User Control — Edit/Hapus Transaksi (Minimal)
**File:** `barang-masuk/page.tsx`, `barang-keluar/page.tsx`
- Tambahkan tombol "Hapus" di tabel transaksi (dengan ConfirmationDialog)
- Jika hapus transaksi masuk: kurangi stok
- Jika hapus transaksi keluar: tambah stok kembali
- **Catatan:** Backend tidak diubah — ini operasi delete + update stok dari client

### 5.3 Flexibility — Keyboard Shortcuts
**File:** `src/components/ui/keyboard-shortcuts.tsx` (BARU)
- `Ctrl+K` atau `/` → Fokus ke search
- `Ctrl+N` → Buka dialog tambah baru
- `Escape` → Tutup dialog
- `?` → Buka panel shortcuts info
- Tambahkan panel "Keyboard Shortcuts" yang bisa diakses dari sidebar

### 5.4 Consistency — Select Component
**File:** `src/components/ui/select.tsx` (sudah ada)
- Ganti semua `<select>` native di form dengan komponen `Select` dari shadcn/ui
- Terapkan di: barang, kategori, supplier, barang-masuk, barang-keluar, laporan, users

### 5.5 Help & Documentation — Onboarding Tooltip
- Tambahkan `data-tooltip` attributes di field yang membingungkan
- Buat komponen `HelpText` untuk deskripsi di bawah field
- Terapkan di form barang (kode, stok minimal, satuan)
